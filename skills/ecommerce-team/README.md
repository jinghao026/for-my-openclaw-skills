## Agent职责详解

### 1️⃣ Product Research Agent - 选品研究专家

- 市场趋势分析
- 竞品深度调研
- 利润测算模型
- 选品风险评估

**输出成果**：
- 选品研究报告
- 利润分析表
- 竞品对比表
- 选品建议清单

---

### 2️⃣ Listing Agent - 产品上架专家

## 典型业务场景

### 🚀 场景1：新品上线全流程

**流程**：
```
Boss: "我要上线一款智能手表"
↓
Coordinator分配任务：
  ├─ Product Research: 竞品调研、定价分析（2天）
  ├─ Listing: 创建Listing、A+页面（1天）
  ├─ Inventory: 准备库存、设置安全库存（并行）
  ├─ Marketing: 制定推广计划（并行）
  └─ Logistics: 准备物流方案（并行）
↓
整合输出 → 完整上线方案 → 执行上线
```

**交付成果**：
- 选品调研报告
- 优化后的Listing
- 库存准备方案
- 营销推广计划
- 物流渠道选择

---

### 📊 场景2：日常运营监控

**流程**：
```
Boss: "查看昨日运营数据"
↓
Coordinator收集：
  ├─ Order Mgmt: 订单处理情况
  ├─ Customer Service: 客服数据
  ├─ Inventory: 库存预警
  ├─ Analytics: 销售和利润数据
  └─ Logistics: 物流状态
↓
整合 → 运营日报 → 向Boss汇报
```

**交付成果**：
- 运营日报
- 异常预警
- 优化建议

---

### 🎉 场景3：节日促销活动

**流程**：
```
Boss: "策划Prime Day促销"
↓
Coordinator协调：
  ├─ Marketing: 促销方案、广告预算
  ├─ Inventory: 促销库存准备
  ├─ Listing: 促销页面、优惠券
  ├─ Customer Service: 客服话术
  └─ Analytics: 目标设定、效果预测
↓
整合 → 促销方案 → 执行 → 效果跟踪
```

**交付成果**：
- 促销活动方案
- 库存准备计划
- 广告投放策略
- 客服应对方案
- 效果跟踪报表

## 使用指南

### 方式一：直接下达业务指令
```
Boss: "我要在Amazon上卖蓝牙耳机，帮我做全套方案"
↓
Coordinator自动分配所有相关Agent
↓
7天后交付完整方案
```

### 方式二：指定具体任务
```
Boss: "让Listing Agent帮我优化产品标题"
↓
Coordinator直接分配任务给Listing Agent
↓
1天内交付优化结果
```

### 方式三：多Agent协作项目
```
Boss: "帮我分析竞品并制定营销策略"
↓
Coordinator分配：
  ├─ Product Research: 竞品分析
  └─ Marketing: 基于竞品制定策略
↓
整合两个Agent输出 → 完整方案
```

### 实时汇报
- 重大异常立即汇报
- 重要节点及时更新
- 任务完成后提交报告

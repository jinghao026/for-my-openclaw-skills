## Agent职责分工

| Agent | 核心职责 | 专长领域 |
|-------|---------|---------|
| **Coding Agent** | 代码编写、Bug修复、技术方案 | Python, JS/TS, CLI工具 |
| **Frontend Design Agent** | UI设计、网页开发、界面美化 | HTML/CSS, React, Vue |
| **Legal Agent** | 合同审查、政策查询、合规检查 | 商业合同、劳动法规 |
| **Customer Service Agent** | 客户咨询、问题解答、工单管理 | 多轮对话、情绪识别 |
| **Finance Agent** | 财务整理、收支记录、报表生成 | 个人财务、基础会计 |

## 工作流程

### 标准工作流程

```
┌─────────────────────────────────────────────────────────────┐
│  1. Boss提出需求                                              │
│     "我需要开发一个财务管理系统"                               │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  2. Coordinator（小秘）分析需求                               │
│     - 需要前端界面 → Frontend Design Agent                    │
│     - 需要后端代码 → Coding Agent                             │
│     - 需要财务逻辑 → Finance Agent                            │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  3. 分配任务给各Agent                                         │
│     - 设定截止时间                                           │
│     - 明确交付标准                                           │
│     - 提供必要信息                                           │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  4. Agent执行 + Coordinator督促                              │
│     - 进度检查（每30分钟）                                    │
│     - 问题解决协调                                           │
│     - 质量把控                                               │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  5. Agent汇报结果                                            │
│     - 按标准格式汇报                                         │
│     - 提供代码/文档/截图                                     │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  6. Coordinator整合结果                                       │
│     - 汇总各Agent输出                                        │
│     - 统一格式风格                                           │
│     - 生成完整方案                                           │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  7. 向Boss汇报                                                │
│     - 完整项目交付                                           │
│     - 使用说明                                               │
│     - 后续建议                                               │
└─────────────────────────────────────────────────────────────┘
```

## 使用指南

### 方式一：直接指定Agent
```
Boss: "让Coding Agent帮我写一个Python脚本"
↓
小秘：立即分配给Coding Agent，跟踪进度，返回结果
```

### 方式二：描述需求，小秘自动分配
```
Boss: "我需要一个网站来展示我的产品"
↓
小秘：分析需求 → 分配Frontend Design Agent + Coding Agent
        → 协调分工 → 整合结果 → 向Boss汇报
```

### 方式三：复杂项目多Agent协作
```
Boss: "开发一个电商管理系统"
↓
小秘：项目拆解
  ├─ Frontend Design Agent: 设计UI界面
  ├─ Coding Agent: 开发前后端代码
  ├─ Finance Agent: 设计财务模块
  └─ Legal Agent: 审核用户协议
        ↓
     小秘整合所有输出 → 完整系统交付
```

## 汇报机制

### Agent向小秘汇报
- **开始执行** - 收到任务后立即确认
- **进度更新** - 每30分钟汇报进度
- **遇到问题** - 立即上报寻求帮助
- **任务完成** - 按标准格式提交结果

### 小秘向Boss汇报
- **任务分配后** - 告知已分配给哪个Agent
- **进行中** - 重要节点汇报进度
- **完成后** - 提交完整结果报告
- **遇问题时** - 立即请示Boss决策

---

## 质量控制

### 交付标准
- ✅ 符合需求描述
- ✅ 代码可正常运行
- ✅ 文档完整清晰
- ✅ 格式规范统一

### 审核流程
1. Agent自检
2. Coordinator（小秘）审核
3. 不通过 → 返回修改
4. 通过 → 整合汇报
